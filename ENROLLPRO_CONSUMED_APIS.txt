ENROLLPRO CONSUMED APIS LIST
============================
Last Updated: 2026-05-15
Source of Truth (SSOT): EnrollPro (Identity & Enrollment)

This document lists the specific EnrollPro APIs that ATLAS, AIMS, and SMART must fetch from to process data within their respective systems.

--------------------------------------------------------------------------------
1. GLOBAL AUTHENTICATION (SSOT for all Credentials)
--------------------------------------------------------------------------------
Consuming Systems: ALL (ATLAS, AIMS, SMART)
Description: All systems use EnrollPro as the identity provider.

- POST /api/auth/login
  * Action: Authenticates users (Teachers, Admins, Registrars).
  * Usage: To obtain the JWT required for protected companion system actions.
- POST /api/auth/verify
  * Action: Verifies credentials without establishing a session.
  * Usage: Rapid credential check for subsystem provisioning.
- POST /api/auth/learner-login
  * Action: Authenticates students using LRN + portal PIN.
  * Usage: Allows learners to access LMS (AIMS) or check grades (SMART).
- GET /api/auth/me
  * Action: Returns current user profile.
  * Usage: To verify role-based permissions in subsystems.

--------------------------------------------------------------------------------
2. ATLAS (Scheduling & Teaching Load)
--------------------------------------------------------------------------------
Consuming Systems: ATLAS
Description: ATLAS requires faculty and section metadata to generate master schedules.

- GET /api/integration/v1/school-year
  * Usage: To sync with the current active academic period.
- GET /api/integration/v1/faculty
  * Usage: SSOT for all teachers. ATLAS fetches this to assign teaching loads.
- GET /api/integration/v1/sections
  * Usage: SSOT for all class sections. ATLAS fetches this to create the timetable.
- GET /api/integration/v1/default/faculty
  * Usage: Optimized feed for active-only faculty syncing.

--------------------------------------------------------------------------------
3. AIMS (LMS & Remediation)
--------------------------------------------------------------------------------
Consuming Systems: AIMS
Description: AIMS requires student enrollment context and remedial flags to assign courses.

- GET /api/integration/v1/default/aims/context
  * Usage: SSOT for LMS enrollment. Fetches LRNs, Sections, and 'isRemedialRequired' flags.
- GET /api/integration/v1/default/faculty
  * Usage: To assign course ownership to the correct teachers.
- GET /api/integration/v1/school-year
  * Usage: To group courses by academic year.

--------------------------------------------------------------------------------
4. SMART (Grading & Academic Records)
--------------------------------------------------------------------------------
Consuming Systems: SMART
Description: SMART requires student rosters for grade encoding and historical reconciliation.

- GET /api/integration/v1/default/smart/students
  * Usage: SSOT for student rosters. Includes ENROLLED, TEMPORARILY_ENROLLED, and DROPPED learners.
- GET /api/integration/v1/sections
  * Usage: To map grade-encoding forms to the correct sections.
- GET /api/integration/v1/school-year
  * Usage: To ensure grades are recorded against the correct historical SY.
- GET /api/integration/v1/sections/:sectionId/learners
  * Usage: Granular fetching of a specific class roster for a teacher's grade sheet.

--------------------------------------------------------------------------------
5. SYSTEM SETTINGS & ASSETS
--------------------------------------------------------------------------------
Consuming Systems: ALL
Description: For consistent branding across the ecosystem.

- GET /api/settings/public
  * Usage: Fetches school name, logo URL, and active accent colors for UI synchronization.
- /uploads/* (Static Assets)
  * Usage: Direct URL fetching of the school logo and student photos.

--------------------------------------------------------------------------------
NOTE ON HISTORICAL DATA:
As of May 15, 2026, all /integration/v1 learner feeds now correctly include 
historical students (e.g., JHS_COMPLETER) when querying past school years. 
SMART and AIMS should always pass the ?schoolYearId= parameter to ensure 
they are fetching the correct population for the academic year they are processing.

A) CONFIRMED INTEGRATION V1 ALIASES AND OPERATIONAL ENDPOINTS

The following routes are currently mounted and available:

  * Usage: System-to-system liveness check (DB + downstream system probe summary).

- GET /api/integration/v1/learners
  * Usage: Enrollment learner feed (requires schoolYearId query in current behavior).

- GET /api/integration/v1/students
  * Usage: Alias of /learners for compatibility with consuming clients.

- GET /api/integration/v1/faculty
  * Usage: Faculty feed.

- GET /api/integration/v1/teachers
  * Usage: Alias of /faculty for compatibility with consuming clients.

- GET /api/integration/v1/staff
  * Usage: Staff feed including configured staff roles.

- GET /api/integration/v1/sections
  * Usage: Section list feed.

- GET /api/integration/v1/sections/:sectionId/learners
  * Usage: Section roster feed.

- GET /api/integration/v1/default/faculty
  * Usage: Default faculty feed for downstream systems.

- GET /api/integration/v1/default/smart/students
  * Usage: SMART default roster feed.

- GET /api/integration/v1/default/aims/context
  * Usage: AIMS default LMS context feed.

--------------------------------------------------------------------------------
B) FEED BEHAVIOR CLARIFICATIONS FOR CONSUMERS
--------------------------------------------------------------------------------

ATLAS:
- Recommended feeds remain:
  * GET /api/integration/v1/school-year
  * GET /api/integration/v1/faculty (or /teachers alias)
  * GET /api/integration/v1/sections
  * GET /api/integration/v1/default/faculty

AIMS:
- Default AIMS context feed currently returns enrolled context entries with
  enrollment records and remedial flag support:
  * GET /api/integration/v1/default/aims/context

SMART:
- Default SMART roster currently includes statuses:
  ENROLLED, TEMPORARILY_ENROLLED, and DROPPED.
  * GET /api/integration/v1/default/smart/students

--------------------------------------------------------------------------------
C) SCHOOL YEAR QUERY RULE (IMPORTANT)
--------------------------------------------------------------------------------

For historical and reconciliation-safe pulls, consumers should pass:
- ?schoolYearId=<id>

This is strongly required for ATLAS/AIMS/SMART synchronization consistency,
especially when processing non-active school years.
