================================================================================
DATA FLOW DICTIONARY & TABLE MAPPING (SENIOR ARCHITECT AUDIT)
================================================================================

PROCESS: P6.0 Verify & Screen Applications
-> Inbound from Registrar: [Document Compliance Submission & Assessment Scores] (Noun: Packet of compliance flags and numerical SCP assessment scores)
<- Inbound from DB: [Pending Application Data] (Reads D10 early_registration_applications to identify the applicant pool)
<- Inbound from DB: [Rubric Criteria] (Reads D22 scp_program_configs and D23 scp_program_steps to validate scores)
<- Inbound from DB: [Prior Compliance Data] (Reads D17 application_checklists and D11 early_registration_assessments to compile report)
<- Outbound to Registrar: [Compliance Verification Report] (Noun: qualified applicants and missing document notifications)
-> Outbound to DB:
   - D17: application_checklists (Persists boolean compliance flags)
   - D10: early_registration_applications (Updates eligibility status)
   - D11: early_registration_assessments (Persists SCP screening exam and interview results)
   - D20: audit_logs (Audit Log Entry: security trail)

PROCESS: P7.0 Manage Sectioning & Enrollment
-> Inbound from Registrar: [Beginning of School Year Sectioning Plan] (Noun: Structured assignment map of approved learners)
<- Inbound from DB: [Pending Enrollment Data] (Reads D12 enrollment_applications to retrieve the learner pool)
<- Inbound from DB: [Learner Data] (Reads D9 learners for demographic verification)
<- Inbound from DB: [Section Capacities] (Reads D6 sections to enforce slot limits)
<- Inbound from DB: [Enrollment Window Status] (Reads D4 school_years to verify BOSY window)
<- Inbound from DB: [TLE Track Requirements] (Reads D8 tle_programs to align G9/G10 learners)
<- Outbound to Registrar: [Official Class Roster & SF1 Report] (Noun: Finalized section class lists)
-> Outbound to DB:
   - D18: enrollment_records (Persists definitive learner-to-section linkage)
   - D12: enrollment_applications (Updates application status to ENROLLED)
   - D6: sections (Updates real-time section capacity metrics)
   - D7: section_advisers (Persists historical section adviser ledger entry)
   - D20: audit_logs (Audit Log Entry: security trail)

PROCESS: P8.0 Finalize EOSY Promotional Status
-> Inbound from Registrar: [EOSY Finalization Data] (Noun: Packet containing general average, academic status, and health measurements)
<- Inbound from DB: [Learner Profile] (Reads D9 learners for student identity validation)
<- Inbound from DB: [Active Class Roster] (Reads D18 enrollment_records and D6 sections for context)
<- Outbound to Registrar: [SF1 Register & SF5 Promotion Master] (Noun: Finalized SF1 class register and SF5 promotion master list)
-> Outbound to DB:
   - D18: enrollment_records (Persists final promotional data)
   - D6: sections (Applies isEosyFinalized lock)
   - D21: health_records (Persists end-of-year student health measurements)
   - D20: audit_logs (Audit Log Entry: security trail)
