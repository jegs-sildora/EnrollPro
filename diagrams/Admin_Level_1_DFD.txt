================================================================================
DATA FLOW DICTIONARY & TABLE MAPPING (SENIOR ARCHITECT AUDIT)
================================================================================

PROCESS: P1.0 Configure School Settings
-> Inbound from Admin: [School Identity & School Year Lifecycle Parameters] (Noun: Metadata packet containing school name, logo, accent color, and academic year window dates)
<- Inbound from DB: [Active Configuration] (Reads D8 school_settings and D9 school_years to populate current state for the status report)
<- Outbound to Admin: [Active School Year Lifecycle Status Summary] (Noun: Report confirming current branding identity and enrollment window state)
-> Outbound to DB:
   - D8: school_settings (Persists singleton school identity: name, logo, accent color)
   - D9: school_years (Persists academic calendar windows and enrollment phase dates)
   - D10: audit_logs (Audit Log Entry: immutable security trail)

PROCESS: P2.0 Manage Users & Access
-> Inbound from Admin: [Personnel Credentials & Role Assignments] (Noun: Security packet with email, employee ID, role, and department assignment)
<- Inbound from DB: [Existing Personnel Data] (Reads from D11 users, D12 teachers, and D21 departments to enforce uniqueness and prevent duplicates)
<- Outbound to Admin: [Personnel Directory] (Noun: Searchable collection of system user accounts and linked teacher profiles)
-> Outbound to DB:
   - D11: users (Persists system account credentials and RBAC role assignments)
   - D12: teachers (Persists teacher profile demographics and contact data)
   - D13: teacher_designations (Persists SY-scoped advisory and ancillary role assignments)
   - D10: audit_logs (Audit Log Entry: security trail for RBAC changes)

PROCESS: P3.0 Configure Special Programs (SCP)
-> Inbound from Admin: [SCP Criteria & Rubrics] (Noun: Configuration packet for STE/SPA admission logic, assessment steps, and scoring rubric criteria)
<- Inbound from DB: [Current Program Logic] (Reads D14 scp_program_configs to retrieve existing pipeline state before applying updates)
<- Outbound to Admin: [Configured Admission Pipelines] (Noun: Confirmation summary of active SCP admission tracks, steps, and rubric weights)
-> Outbound to DB:
   - D14: scp_program_configs (Persists SCP pipeline formula, steps, rubric categories, and admission options)
   - D10: audit_logs (Audit Log Entry: security trail for eligibility logic changes)

PROCESS: P9.0 Finalize School Year
-> Inbound from Admin: [School Year Closing Parameters] (Noun: Trigger parameters for global school year archival and LIS export generation)
<- Inbound from DB: [Promotional Records] (Reads D17 enrollment_records for final promotion status and general averages)
<- Inbound from DB: [School Year Context Data] (Reads D1 learners, D12 teachers, and D18 sections for DepEd LIS compliance)
<- Outbound to Admin: [LIS Export Package & Completion Report] (Noun: Final DepEd LIS CSV export package and statutory school year completion report)
-> Outbound to DB:
   - D9: school_years (Updates school year status to ARCHIVED; isEosyFinalized = true)
   - D10: audit_logs (Audit Log Entry: security trail for year-end archival)
