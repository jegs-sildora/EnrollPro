================================================================================
DATA FLOW DICTIONARY & TABLE MAPPING (SENIOR ARCHITECT AUDIT)
================================================================================

PROCESS: P4.0 Submit Early Registration
-> Inbound from Learner: [BEEF Annex 1 Registration Packet] (Noun: DepEd BEEF Annex 1 form data: LRN, full name, birthdate, contact info, and mother tongue)
<- Inbound from DB: [Enrollment Window Status] (Reads D4 school_years to verify Phase 1 Early Registration window is open)
<- Inbound from DB: [Grade Hierarchy Reference] (Reads D5 grade_levels for JHS hierarchy validation)
<- Outbound to Learner: [Phase 1 Tracking Number] (Noun: 10-digit alphanumeric monitoring identifier assigned to the Early Registration Application)
-> Outbound to DB:
   - D9: learners (Persists permanent learner identity: LRN, name, birthdate, mother tongue)
   - D10: early_registration_applications (Persists Phase 1 Early Registration metadata)
   - D15: application_addresses (Persists current and permanent geographic data)
   - D16: application_family_members (Persists parent and guardian profiles)
   - D20: audit_logs (Audit Log Entry: security trail)

PROCESS: P5.0 Submit Official Enrollment
-> Inbound from Learner: [BEEF Annex 2 Enrollment Packet] (Noun: DepEd BEEF Annex 2 form data: learning modality, intake method, and SCP track specialization)
<- Inbound from DB: [Enrollment Window Status] (Reads D4 school_years to verify Phase 2 Official Enrollment window is open)
<- Inbound from DB: [Pre-populated Profile] (Reads D9 learners, D15 application_addresses, D16 application_family_members to eliminate redundant data entry)
<- Inbound from DB: [TLE Track Catalog] (Reads D8 tle_programs for specialization selection in G9/G10)
<- Outbound to Learner: [Enrollment Application Status] (Noun: Notification of official enrollment placement state: PENDING_REVIEW or QUALIFIED)
-> Outbound to DB:
   - D12: enrollment_applications (Persists official BOSY enrollment intent and intake method)
   - D13: enrollment_previous_schools (Persists SF10 previous school history)
   - D14: enrollment_program_details (Persists SCP track and strand specifics)
   - D20: audit_logs (Audit Log Entry: security trail)

PROCESS: P5.1 Submit Continuing Learner Confirmation
-> Inbound from Learner: [Return Intent Packet] (Noun: Returning Grade 8-10 learner consent submission for BOSY re-enrollment)
<- Inbound from DB: [Existing Application] (Reads D12 enrollment_applications to link the confirmation to the learner's prior profile)
<- Outbound to Learner: [Printed Confirmation Slip] (Noun: Printable verification document confirming return intent with intakeMethod = CONFIRMATION_SLIP)
-> Outbound to DB:
   - D12: enrollment_applications (Updates intake method to CONFIRMATION_SLIP and sets status to PENDING_REVIEW)
   - D20: audit_logs (Audit Log Entry: security trail)

PROCESS: P5.2 Monitor Application Status & View Placement
-> Inbound from Learner: [Portal Credentials] (Noun: LRN and PIN, or Tracking Number for Phase 1 status checks)
<- Inbound from DB: [Application Status] (Reads D10 early_registration_applications or D12 enrollment_applications to retrieve current state)
<- Inbound from DB: [Section/Adviser Data] (Reads D18 enrollment_records, D6 sections, and D2 teachers to show finalized placement)
<- Outbound to Learner: [Portal Dashboard View] (Noun: Digital display of enrollment status, document compliance flags, and class adviser info)
-> Outbound to DB:
   - D20: audit_logs (Audit Log Entry: records portal access for security monitoring)
