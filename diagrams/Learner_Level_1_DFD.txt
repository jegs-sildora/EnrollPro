================================================================================
DATA FLOW DICTIONARY & TABLE MAPPING (DEPED CONCISE)
================================================================================

PROCESS: P4.0 Submit Early Registration
-> Inbound from Actor: [IN] BEEF Annex 1 Profile (LRN, demographics, and contact info)
<- Outbound to Actor: [OUT] Phase 1 Tracking No. (10-digit monitoring key)
-> Outbound to DB:
   - D1: learners (Persists permanent identity attributes)
   - D2: early_registration_applications (Persists Phase 1 metadata and SY-status)
   - D3: application_addresses (Persists current/permanent geographic data)
   - D4: application_family_members (Persists parent and guardian profiles)

PROCESS: P5.0 Submit Official Enrollment
-> Inbound from Actor: [IN] BEEF Annex 2 Modalities (Modality selection, history, and tracks)
-> Inbound from DB: [OUT from DB] Pre-populated Profile (Retrieves D1-D4 to prevent re-entry)
<- Outbound to Actor: [OUT] Official Enrollment Status (Placement results notification)
-> Outbound to DB:
   - D5: enrollment_applications (Persists official Phase 2 intent and modalities)
   - D6: enrollment_previous_schools (Persists SF10 academic history for records)
   - D7: enrollment_program_details (Persists SCP-specific track specializations)

PROCESS: P5.1 Submit Continuing Learner Confirmation
-> Inbound from Actor: [IN] Existing Student ID & Return Intent (Continuing learner consent slip)
<- Outbound to Actor: [OUT] Confirmation Status (Immediate system feedback)
-> Outbound to DB:
   - D5: enrollment_applications (Inserts record via intakeMethod='CONFIRMATION_SLIP')
